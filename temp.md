아래는 **`jetson-main(4).zip` 기준**으로,
**`UiTest.zip` / `yolo_test-main (1).zip`에 이미 있던 기능은 제외**하고 정리한 목록입니다.

## 사용자 입장에서 새롭게 구현된 기능

| 구분                    | 구현된 기능                                  | 사용자 입장에서 보이는 의미                     |
| --------------------- | --------------------------------------- | ----------------------------------- |
| **실시간 관제 통합**         | PyQt UI가 실제 카메라/AI 파이프라인과 연결됨           | 단순 테스트 UI가 아니라, 실제 탐지 결과가 화면에 반영됨   |
| **전체 위험 상태 표시**       | 상단 경고 바에 전체 카메라 중 가장 높은 위험 단계 표시        | 여러 카메라 중 하나라도 위험하면 화면 상단에서 바로 확인 가능 |
| **다중 카메라 실시간 연동**     | 여러 카메라를 동시에 백그라운드에서 캡처                  | 화면에 보이는 카메라만이 아니라, 모든 카메라가 동시에 감시됨  |
| **분할 화면 관제**          | 여러 카메라 영상을 한 화면에 분할 표시                  | 운전자가 여러 방향을 동시에 확인 가능               |
| **카메라 화면 확대**         | 분할 화면 중 하나를 클릭/터치하면 확대 표시               | 특정 카메라 화면을 크게 볼 수 있음                |
| **카메라별 ROI 설정 저장**    | 카메라마다 위험 구역 ROI를 JSON으로 저장/불러오기         | 카메라마다 다른 위험 구역을 유지 가능               |
| **ROI 설정 안정화**        | ROI 그리기를 PyQt Painter 대신 OpenCV 방식으로 처리 | ROI 설정 화면에서 충돌/멈춤 가능성을 줄임           |
| **실제 이벤트 로그 표시**      | 이벤트 화면이 더미 데이터가 아니라 실제 로그 파일을 읽음        | 감지된 위험 이벤트 기록을 UI에서 확인 가능           |
| **시스템 종료 버튼**         | 설정 화면에서 프로그램 종료 가능                      | 키보드 없이 터치 UI로 시스템 종료 가능             |
| **GPIO 부저 연동**        | 위험 감지 시 Jetson GPIO 기반 부저 제어            | 화면뿐 아니라 실제 경고음/부저로 알림 가능            |
| **카메라 연결 재시도**        | 카메라 연결 실패 시 자동 재시도                      | 일시적인 연결 문제에 더 강함                    |
| **카메라 오류 진단**         | 장치 없음, 사용 중, 백엔드 문제 등을 구분               | 문제 발생 시 원인 파악이 쉬움                   |
| **이벤트 기반 영상 녹화**      | 위험 발생 시 자동 녹화 시작                        | 위험 상황만 증거 영상으로 저장 가능                |
| **사전 버퍼 녹화**          | 위험 감지 전 일정 시간의 영상도 함께 저장                | “위험이 발생하기 전 상황”까지 확인 가능             |
| **후속 녹화**             | 위험이 끝난 뒤에도 일정 시간 추가 저장                  | 위험 종료 직후 상황까지 증거로 남길 수 있음           |
| **모든 카메라 동시 녹화**      | 한 카메라에서 위험이 감지되면 전체 카메라 영상 저장           | 사고 상황을 여러 각도에서 확보 가능                |
| **상시 녹화 모드**          | 이벤트 녹화 외에 full recording 모드 지원          | 필요하면 블랙박스처럼 계속 녹화 가능                |
| **이벤트별 폴더 저장**        | 위험 이벤트마다 별도 폴더 생성                       | 나중에 사건 단위로 영상 확인이 쉬움                |
| **H.264 MP4 저장**      | ffmpeg/libx264 기반 MP4 저장                | 저장 영상 호환성과 업로드 적합성이 좋아짐             |
| **영상 서버 업로드**         | 저장된 이벤트 영상을 서버로 업로드                     | Jetson 안에만 저장하지 않고 서버에 증거 영상 전송 가능  |
| **업로드 재시도/타임아웃 처리**   | 영상 업로드 실패 시 재시도                         | 네트워크가 불안정해도 업로드 안정성 향상              |
| **이벤트 JSON 전송 워커**    | 위험 이벤트 메타데이터를 별도 스레드에서 전송               | AI 추론/화면 표시를 막지 않고 서버에 이벤트 정보 전송    |
| **중복 이벤트 전송 방지**      | 카메라별 쿨다운 적용                             | 같은 위험 상황이 서버에 너무 많이 중복 전송되는 것을 방지   |
| **GPS 연동 구조**         | GPS 데이터를 읽고 영상/이벤트 정보에 붙일 수 있음          | 사고 위치 정보를 증거 데이터에 포함 가능             |
| **GPS 좌표 기반 파일명/폴더명** | GPS가 있으면 저장 파일/폴더에 위치 정보 반영             | 나중에 어느 위치에서 발생한 이벤트인지 추적 가능         |
| **IMU 연동 구조**         | IMU 센서 초기화 및 읽기 구조 존재                   | 충격/움직임 데이터 확장을 위한 기반이 있음            |
| **센서 데이터 버퍼링**        | GPS/IMU 데이터를 최신값 기준으로 보관                | 영상 시점과 센서 데이터를 함께 묶을 수 있음           |
| **구조화 로그**            | 이벤트/오류/상태 로그를 분류해서 기록                   | 디버깅, 발표 자료, 사고 기록 분석에 활용 가능         |
| **성능 로그**             | 추론 시간, 후처리 시간, 탐지 수 등을 주기적으로 기록         | Jetson에서 어느 부분이 느린지 확인 가능           |
| **Watchdog 자동 재시작**   | 프로그램 비정상 종료 시 재시작 가능                    | 현장 장비처럼 계속 켜져 있어야 하는 상황에 유리         |
| **부팅 자동 실행 설정 기반**    | systemd 자동 실행 설정 템플릿 존재                 | Jetson 부팅 후 자동으로 관제 프로그램 실행 가능      |
| **저장 공간 관리 기능**       | 오래된 파일/폴더 정리 기능 존재                      | 저장 공간 부족으로 시스템이 멈추는 상황을 줄일 수 있음     |
| **안전한 종료/정리 절차**      | 종료 시 카메라, 부저, 센서, 저장 스레드 정리             | 파일 깨짐이나 장치 점유 문제를 줄임                |

## 중복으로 보고 제외한 기능

아래 기능들은 `UiTest.zip` 또는 `yolo_test-main (1).zip`에도 이미 있어서, 위 목록에서는 **핵심 기능으로는 제외**했습니다.

| 제외한 기능                                      | 이유                                           |
| ------------------------------------------- | -------------------------------------------- |
| 기본 PyQt 화면 구성                               | `UiTest.zip`에 메뉴, 라이브, 재생, 설정, ROI 화면이 이미 있음 |
| 기본 영상 재생 UI                                 | `UiTest.zip`에 녹화 목록/재생 팝업이 이미 있음             |
| 기본 ROI 설정 화면                                | `UiTest.zip`에 ROI 설정 UI가 이미 있음               |
| 기본 YOLO 사람 탐지                               | `yolo_test-main`에 이미 있음                      |
| ROI 내부 사람 감지                                | `yolo_test-main`에 이미 있음                      |
| 발끝 기준 ROI 판정                                | `yolo_test-main`에 이미 있음                      |
| 동적 ROI 개념                                   | `yolo_test-main/main_system.py`에 이미 있음       |
| TTC/박스 면적 증가 기반 위험 단계                       | `yolo_test-main/main_system.py`에 이미 있음       |
| 위험 단계 SAFE / BLIND_SPOT / APPROACH / URGENT | `yolo_test-main`에 이미 있음                      |
| 단순 서버 JSON 이벤트 전송 개념                        | `yolo_test-main`에 이미 있음                      |

## 발표용으로 짧게 말하면

`jetson-main(4)`에서 새롭게 강조할 수 있는 기능은 이렇게 정리하면 좋습니다.

> 기존의 단순 UI 테스트와 YOLO 단독 테스트를 통합하여,
> **Jetson에서 실제 다중 카메라 관제, 위험 이벤트 기반 증거 영상 저장, 서버 업로드, GPS/IMU 센서 연동, GPIO 부저 경고, 로그/Watchdog/저장공간 관리까지 포함한 현장형 안전 블랙박스 시스템**으로 확장했다.

특히 사용자 입장에서 가장 의미 있는 차별점은:

1. **위험 발생 시 여러 카메라 영상을 동시에 증거로 저장**
2. **감지 전/후 영상까지 포함하는 이벤트 녹화**
3. **저장된 영상을 서버로 자동 업로드**
4. **GPS 위치 정보와 이벤트 로그를 함께 관리**
5. **Jetson 장비에서 장시간 운용하기 위한 재시작/로그/저장공간 관리 구조**

입니다.
